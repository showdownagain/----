# MT5 Python 自动交易与看盘系统完整实施方案

## 一、项目目标

本项目目标是在本地 Windows 11 电脑上搭建一套可运行的 MT5 交易辅助系统。

系统包括：

1. 使用本地 MetaTrader 5 Terminal 连接券商账户。
2. 使用 Python 调用 MetaTrader5 官方 Python 库获取行情、账户、持仓和订单数据。
3. 使用 Python 编写交易策略。
4. 使用 FastAPI 提供后端 API 和 WebSocket 实时推送。
5. 使用 Vue 或 React + Lightweight Charts 构建前端看盘界面。
6. 使用 SQLite 保存交易日志、策略信号、告警记录和系统状态。
7. 使用 Telegram、飞书、钉钉或邮件发送交易提醒。
8. 后期可迁移到 Windows 云服务器 24 小时运行。

MetaTrader 官方 Python 集成文档说明，MetaTrader5 Python 包通过进程间通信从本地 MetaTrader 5 Terminal 获取交易数据，因此本方案的核心是：**Python 不直接连接券商 MT5 服务器，而是通过本地运行的 MT5 Terminal 作为桥梁**。

## 三、技术选型

### 1. 交易接口

使用：

```text
MetaTrader5 Python 官方库
```

用途：

```text
连接本地 MT5 Terminal
获取账户信息
获取 Tick 行情
获取历史 K 线
获取当前持仓
检查订单
发送订单
```

官方文档中提供了 `copy_rates_from_pos` 获取 K 线数据，返回包含 time、open、high、low、close、tick_volume、spread、real_volume 等字段的数据。
官方文档中也提供了 `order_send`，用于从终端向交易服务器发送交易请求。

### 2. 后端框架

使用：

```text
FastAPI
```

FastAPI 是基于 Python 类型提示构建 API 的现代 Web 框架，适合做交易系统的 REST API、WebSocket、账户接口和策略接口。

实时推送使用：

```text
FastAPI WebSocket
```

FastAPI 官方文档提供了 WebSocket 示例，可用于浏览器和后端之间保持持续连接，适合推送实时价格、订单状态和策略信号。

### 3. 前端框架

可选：

```text
Vue 3 + Vite
```

或者：

```text
React + Vite
```

图表使用：

```text
Lightweight Charts
```

Lightweight Charts 是 TradingView 官方开源的轻量图表库，适合构建高性能 K 线图和行情图表。
官方示例也展示了通过 `series.update(...)` 实现 K 线实时更新。

### 4. 数据库

第一版建议：

```text
SQLite
```

原因：

```text
不需要单独安装数据库服务
适合本地开发
备份简单
方便迁移
```

后期正式运行可以升级：

```text
PostgreSQL for Windows
SQL Server Express
```

## 六、MT5 连接模块

创建：

```text
backend/app/mt5_client.py
```

示例结构：

```python
import MetaTrader5 as mt5
import pandas as pd
from datetime import datetime


class MT5Client:
    def __init__(self):
        self.connected = False

    def connect(self):
        if not mt5.initialize():
            raise RuntimeError(f"MT5 initialize failed: {mt5.last_error()}")

        self.connected = True
        return True

    def shutdown(self):
        mt5.shutdown()
        self.connected = False

    def account_info(self):
        account = mt5.account_info()
        if account is None:
            raise RuntimeError(f"account_info failed: {mt5.last_error()}")
        return account._asdict()

    def positions_get(self, symbol=None):
        if symbol:
            positions = mt5.positions_get(symbol=symbol)
        else:
            positions = mt5.positions_get()

        if positions is None:
            return []

        return [p._asdict() for p in positions]

    def symbol_tick(self, symbol: str):
        tick = mt5.symbol_info_tick(symbol)
        if tick is None:
            raise RuntimeError(f"symbol_info_tick failed: {mt5.last_error()}")
        return tick._asdict()

    def get_rates(self, symbol: str, timeframe, count: int = 200):
        rates = mt5.copy_rates_from_pos(symbol, timeframe, 0, count)
        if rates is None:
            raise RuntimeError(f"copy_rates_from_pos failed: {mt5.last_error()}")

        df = pd.DataFrame(rates)
        df["time"] = pd.to_datetime(df["time"], unit="s")
        return df

    def order_check(self, request: dict):
        result = mt5.order_check(request)
        if result is None:
            raise RuntimeError(f"order_check failed: {mt5.last_error()}")
        return result._asdict()

    def order_send(self, request: dict):
        result = mt5.order_send(request)
        if result is None:
            raise RuntimeError(f"order_send failed: {mt5.last_error()}")
        return result._asdict()
```

### 示例：均线交叉策略

创建：

```text

```

```python
def generate_signal(df, fast=20, slow=60):
    if len(df) < slow + 2:
        return {
            "signal": "HOLD",
            "reason": "not enough bars"
        }

    df = df.copy()
    df["ma_fast"] = df["close"].rolling(fast).mean()
    df["ma_slow"] = df["close"].rolling(slow).mean()

    prev_fast = df["ma_fast"].iloc[-2]
    prev_slow = df["ma_slow"].iloc[-2]
    curr_fast = df["ma_fast"].iloc[-1]
    curr_slow = df["ma_slow"].iloc[-1]

    if prev_fast <= prev_slow and curr_fast > curr_slow:
        return {
            "signal": "BUY",
            "reason": f"MA{fast} crossed above MA{slow}"
        }

    if prev_fast >= prev_slow and curr_fast < curr_slow:
        return {
            "signal": "SELL",
            "reason": f"MA{fast} crossed below MA{slow}"
        }

    return {
        "signal": "HOLD",
        "reason": "no crossover"
    }
```

## 八、风控模块设计

创建：

```text
backend/app/risk_service.py
```

风控规则必须放在下单前。

### 必须包含的风控项

```text
1. 是否允许真实交易
2. 单笔最大手数限制
3. 最大持仓数量限制
4. 同品种是否已有持仓
5. 当前点差是否过大
6. 今日亏损是否超过限制
7. 是否在允许交易时间内
8. 是否重复信号
9. 是否通过 MT5 order_check
```

### 风控流程

```text
策略产生信号
    ↓
检查 ENABLE_REAL_TRADING
    ↓
检查品种是否允许交易
    ↓
检查手数是否超限
    ↓
检查当前点差
    ↓
检查当前持仓
    ↓
检查今日亏损
    ↓
调用 order_check
    ↓
允许 / 拒绝交易
```

### 示例逻辑

```python
class RiskService:
    def __init__(self, max_lot=0.05, max_positions=3, enable_real_trading=False):
        self.max_lot = max_lot
        self.max_positions = max_positions
        self.enable_real_trading = enable_real_trading

    def validate_order(self, symbol, lot, positions):
        if not self.enable_real_trading:
            return False, "real trading disabled"

        if lot > self.max_lot:
            return False, "lot exceeds max_lot"

        if len(positions) >= self.max_positions:
            return False, "positions exceed max_positions"

        return True, "ok"
```

### 市价买入请求示例

```python
import MetaTrader5 as mt5


def build_buy_request(symbol, lot, price, sl=None, tp=None, deviation=20):
    request = {
        "action": mt5.TRADE_ACTION_DEAL,
        "symbol": symbol,
        "volume": lot,
        "type": mt5.ORDER_TYPE_BUY,
        "price": price,
        "deviation": deviation,
        "magic": 20260101,
        "comment": "python strategy buy",
        "type_time": mt5.ORDER_TIME_GTC,
        "type_filling": mt5.ORDER_FILLING_IOC,
    }

    if sl:
        request["sl"] = sl

    if tp:
        request["tp"] = tp

    return request
```

```text
前端 / 策略发出下单请求

风控模块检查
    ↓
mt5.order_check(request)
    ↓
mt5.order_send(request)
    ↓
保存数据库
    ↓
推送 WebSocket
    ↓
发送 Telegram / 飞书 / 钉钉提醒
```

### FastAPI 示例

```python
from fastapi import FastAPI
import MetaTrader5 as mt5
from app.mt5_client import MT5Client

app = FastAPI(title="MT5 Trading System")

mt5_client = MT5Client()


@app.on_event("startup")
def startup():
    mt5_client.connect()


@app.on_event("shutdown")
def shutdown():
    mt5_client.shutdown()


@app.get("/api/health")
def health():
    return {"status": "ok"}


@app.get("/api/account")
def get_account():
    return mt5_client.account_info()


@app.get("/api/positions")
def get_positions():
    return mt5_client.positions_get()


@app.get("/api/tick/{symbol}")
def get_tick(symbol: str):
    return mt5_client.symbol_tick(symbol)


@app.get("/api/rates/{symbol}")
def get_rates(symbol: str, count: int = 200):
    df = mt5_client.get_rates(symbol, mt5.TIMEFRAME_M1, count)
    return df.to_dict(orient="records")
```

### Tick 推送示例

```python
from fastapi import WebSocket
import asyncio


@app.websocket("/ws/ticks/{symbol}")
async def websocket_ticks(websocket: WebSocket, symbol: str):
    await websocket.accept()

    try:
        while True:
            tick = mt5_client.symbol_tick(symbol)
            await websocket.send_json(tick)
            await asyncio.sleep(1)
    except Exception:
        await websocket.close()
```

## 十七、安全原则

### 1. 本地开发不要暴露公网

开发阶段只监听：

```text
127.0.0.1
```

不要使用：

```text
0.0.0.0
```

不要把交易控制台直接暴露到公网。

### 2. 实盘下单必须加开关

配置文件中必须有：

```env
ENABLE_REAL_TRADING=false
```

只有你确认进入 Demo 或实盘测试时，才手动改成：

```env
ENABLE_REAL_TRADING=true
```

### 3. 必须有一键停止

前端必须有：

```text
停止所有策略
禁止自动下单
关闭所有 WebSocket 推送
```

### 4. 必须有最大亏损限制

例如：

```text
单日最大亏损：100 美元
单笔最大亏损：20 美元
最大持仓数量：3
单品种最大持仓：1
最大手数：0.05
```

### 5. 防重复下单

策略信号必须记录：

```text
上一次信号时间
上一次信号方向
上一次下单 ticket
当前是否已有持仓
```

同一根 K 线内不要重复下单。

# 结论

你的系统完全可以在本地 Win11 上实施。

最推荐的路线是：

```text
Win11 本地电脑
    ↓
MT5 Terminal 登录券商账户
    ↓
Python MetaTrader5 库读取行情和账户
    ↓
Python 策略生成交易信号
    ↓
FastAPI 提供 API 和 WebSocket
    ↓
Vue / React + Lightweight Charts 展示看盘界面
    ↓
风控模块确认后再允许下单
    ↓
Telegram / 飞书 / 钉钉发送提醒
```

第一版的核心目标不是自动交易，而是：

```text
先把数据链路跑通
再把看盘界面做出来
再把策略信号做出来
再做提醒
最后才做下单
```

这样最稳，也最适合你从本地开发逐步过渡到 Windows 云服务器长期运行。
